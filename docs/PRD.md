# 产品需求文档 (PRD) - SingerOS

> **最后更新：2026-04-21**
> **版本：2.0**


## 1. 产品愿景

构建一个企业级数字员工平台，让企业可以像管理真实员工一样，创建、配置、授权、调度和审计 AI 数字员工。

数字员工不是单纯的聊天机器人。它需要有独立身份、接收任务的入口、真实执行工作的环境，以及模型、工具、技能、知识库等基础能力。

**核心价值：**

- 让 AI 助手融入企业现有工作流程，而非要求员工学习新工具
- 通过标准化配置和管理，确保 AI 行为可控、可审计、可追溯
- 支持多场景、多渠道、多能力的灵活组合，满足企业多样化需求


## 2. 核心概念

### 2.1 数字员工的组成

一个可工作的数字员工由四部分组成：

```
数字员工 = 员工档案 + 绑定关系 + 能力配置 + 运行环境
```

**员工档案：** 定义它是谁、属于哪个部门、负责什么岗位、职责范围是什么。

**绑定关系：** 解决用户和系统如何找到它、它在哪里执行工作的问题。包括：
- IM 身份：在企业通讯工具中的身份
- 外部连接：接收外部系统通知的入口
- 工作节点：执行真实任务的工作环境

**能力配置：** 定义它能使用哪些模型、工具、技能和知识库。

**运行环境：** 定义它如何理解任务、拆解任务、调用能力、执行并返回结果。

### 2.2 绑定关系架构

绑定关系是数字员工能进入企业工作流的关键：

```
数字员工
    |
    +-- IM 身份
    |     用户可以在 IM 中像找同事一样找它
    |
    +-- 外部连接
    |     外部系统可以通过通知、回调等方式告诉它发生了什么事
    |
    +-- 工作节点
          它可以在受控环境中执行编码、命令、文件操作等真实工作
```

**IM 身份**用于绑定独立的即时通讯机器人身份。绑定后，该员工会在 IM 中以自己的名字和头像出现，用户可以直接与它对话。

**外部连接**解决的是"系统如何把事情交给数字员工"的问题。例如代码平台的通知、业务系统告警、工单系统回调等。

**工作节点**解决的是"数字员工在哪里真正干活"的问题。绑定工作节点后，数字员工可以在受控环境中执行命令、修改文件、运行测试、分析代码、生成结果。这相当于给数字员工准备了一台受控的工作电脑。

### 2.3 平台能力分层

平台能力从下到上分为四层：

**基础能力层：** 提供 AI 模型、工作节点、工具技能、知识库、权限、审计等基础支撑。

**执行引擎层：** 负责任务理解、任务拆解、会话管理、能力调用、沙箱调度等核心执行逻辑。

**入口接入层：** 提供 IM 身份、外部系统回调、业务系统接入等多种接入方式。

**业务使用层：** 提供数字员工创建维护、工作台、任务管理、定时任务、多员工协作、技能市场等业务功能。


## 3. 数字员工管理

数字员工管理是平台的核心业务功能，支持创建、配置、启用、停用数字员工。

### 3.1 基础信息

- 名称：数字员工的显示名称
- 头像：数字员工的头像图片
- 描述：数字员工的职责和能力说明
- 部门：数字员工所属的部门
- 岗位：数字员工的岗位角色
- 负责人：管理该数字员工的企业用户
- 启用状态：启用或停用该数字员工

### 3.2 角色配置

**职责范围：** 定义数字员工负责的工作领域，例如"代码审查"、"问题解答"、"数据分析"等。

**工作边界：** 定义数字员工不能做的事情，例如"不允许修改生产环境配置"、"不允许访问敏感数据"等。

**系统提示词：** 定义数字员工的行为准则和回复风格。

**输出格式：** 定义数字员工输出结果的格式要求，例如结构化数据、报告模板等。

**拒绝策略：** 当遇到超出职责范围或违反边界的请求时，数字员工应该如何回应。

**需要人工审批的场景：** 定义哪些操作需要人工审批后才能执行，例如修改重要文件、执行高风险命令等。

### 3.3 能力配置

**默认模型：** 数字员工默认使用的 AI 模型。

**可用工具：** 数字员工可以调用的外部工具，例如代码查询、文件操作、网络请求等。

**可用技能：** 数字员工可以使用的预定义技能，例如代码审查、文本摘要、分类判断等。

**可用外部服务：** 数字员工可以调用的外部服务能力。

**可用知识库：** 数字员工可以查询的企业知识库，用于获取内部上下文信息。

**默认工作节点：** 数字员工执行任务时默认使用的工作环境。

### 3.4 权限配置

**调用权限：** 哪些用户可以调用该数字员工。可以按用户、部门、角色进行配置。

**知识库访问权限：** 数字员工可以访问哪些知识库。

**工具执行权限：** 数字员工可以执行哪些工具，特别是涉及写操作的工具。

**网络访问权限：** 数字员工是否允许访问外部网络。

**写入权限：** 数字员工是否允许修改外部系统的数据，例如提交代码、创建工单等。


## 4. 绑定关系管理

### 4.1 IM 身份

IM 身份用于将数字员工绑定到企业即时通讯工具中的机器人。

**支持的 IM 渠道：**

- 企业微信
- 飞书
- 钉钉
- Slack
- 其他企业 IM 平台

**典型能力：**

- 私聊消息：用户可以直接与数字员工私聊
- 群聊消息：数字员工可以加入群聊并参与讨论
- @ 消息：用户可以 @ 数字员工来触发任务
- 卡片消息：数字员工可以发送结构化的卡片消息
- 文件消息：用户可以发送文件给数字员工处理
- 身份映射：将 IM 用户映射为平台内部用户
- 消息权限校验：验证发送者是否有权限调用数字员工
- 消息审计：记录所有 IM 消息用于审计

**典型状态：**

- 未绑定：无法在 IM 中直接找到该员工
- 已绑定：员工拥有独立 IM 身份，用户可直接对话
- 异常：机器人凭证失效、回调配置错误或渠道不可用

**页面展示内容：**

- 渠道名称
- 机器人名称和头像
- 绑定状态
- 最近消息时间
- 操作入口：绑定、解绑、测试连接

### 4.2 外部连接

外部连接用于接收外部系统推送的通知，解决"系统如何把事情交给数字员工"的问题。

**支持的外部系统：**

- 代码平台：GitHub、GitLab、Gitee 等
- 事件通知：Webhook、API 回调等
- 业务系统：工单系统、审批系统、告警系统等
- 企业内部系统：通过 API 接入的自有系统

**典型通知类型：**

- 代码平台：PR/MR 创建、PR/MR 更新、代码提交、Issue 创建、评论新增等
- 告警系统：监控告警触发、告警恢复等
- 工单系统：工单创建、工单状态变更、审批流变更等
- 自定义事件：外部系统通过 API 创建的任务

**典型能力：**

- 事件类型配置：选择订阅哪些类型的事件
- 请求签名和密钥管理：验证请求来源的合法性
- 内容映射：将外部事件内容映射为平台内部任务
- 幂等处理：避免重复处理同一事件
- 失败重试：事件处理失败时自动重试
- 异步回调：支持长时间任务的异步结果回写
- 触发指定员工：按事件类型触发指定的数字员工

**页面展示内容：**

- 来源系统名称
- 已订阅的事件类型
- 连接健康状态（正常/异常）
- 最近触发时间
- 操作入口：配置、禁用、测试事件

### 4.3 工作节点

工作节点用于绑定数字员工的远程工作环境，解决"数字员工在哪里真正干活"的问题。

**支持的节点类型：**

- 容器沙箱节点：基于容器技术的隔离执行环境
- 远程执行节点：连接到远程服务器执行任务
- 后续可扩展其他类型的执行环境

**典型能力：**

- 节点健康检查：定期检查节点是否可用
- 资源容量管理：监控节点的 CPU、内存、磁盘等资源
- 工作目录管理：管理任务执行的工作目录、仓库目录、缓存目录、结果目录
- 资源限制：限制任务可使用的 CPU、内存、磁盘、网络、执行时长
- 执行审计：记录命令执行日志、文件变更、退出码、执行产物

**典型状态：**

- 未绑定：只能做纯问答或轻量工具调用，无法执行代码类任务
- 已绑定：可以执行代码类、文件类、自动化类任务
- 运行中：节点正在处理任务
- 异常：节点不可用，任务需要暂停、迁移或失败重试

**页面展示内容：**

- 节点名称
- 节点类型
- 运行状态
- 当前任务数
- 资源规格（CPU、内存、磁盘）
- 操作入口：管理节点、切换节点、查看日志

### 4.4 绑定关系通用能力

所有类型的绑定关系都需要支持以下通用能力：

- 创建绑定：将数字员工与外部系统建立连接
- 修改绑定：更新绑定的配置参数
- 启用和停用：临时启用或停用某个绑定
- 测试连接：测试绑定是否正常工作
- 健康检查：定期检查绑定的健康状态
- 变更审计：记录绑定关系的变更历史
- 权限校验：验证操作者是否有权限修改绑定


## 5. 任务执行和管理

### 5.1 任务生命周期

任务从创建到完成经历以下状态：

```
创建 -> 排队 -> 执行中 -> 等待审批 -> 完成
                        -> 失败
                        -> 取消
```

**创建：** 任务被创建，等待分配和执行。

**排队：** 任务已分配给数字员工，等待执行资源。

**执行中：** 数字员工正在处理任务。

**等待审批：** 任务执行到需要人工审批的节点，暂停等待审批。

**完成：** 任务成功完成。

**失败：** 任务执行失败，可能由于错误、超时等原因。

**取消：** 任务被用户或系统取消。

### 5.2 任务属性

**优先级：** 任务的优先级，影响执行顺序。

**截止时间：** 任务需要完成的时间要求。

**标签：** 用于分类和筛选任务的标签。

**关联项目：** 任务关联的项目或业务场景。

**子任务：** 一个任务可以拆分为多个子任务。

**依赖关系：** 任务之间的依赖关系，某些任务需要等待其他任务完成后才能执行。

**人工评论：** 用户可以在任务中添加评论。

**人工审批：** 需要人工审批的任务节点。

**任务验收：** 任务完成后的人工验收确认。

### 5.3 定时任务

定时任务用于周期性执行数字员工工作。

**典型场景：**

- 每日项目进展摘要
- 每周代码仓库巡检
- 定时同步知识库
- 定时分析业务数据
- 定时生成运营报告

**功能范围：**

- Cron 表达式配置：灵活配置调度时间
- 绑定数字员工：指定执行任务的数字员工
- 绑定任务模板：预设任务内容和参数
- 失败重试：任务失败时自动重试
- 错过补偿：如果错过执行时间，是否补偿执行
- 执行记录：记录每次定时任务的执行结果
- 通知和告警：任务执行异常时通知相关人员

### 5.4 多员工协作

复杂任务可以由多个数字员工协作完成。

**典型模式：**

- 主从模式：主员工拆解任务，子员工分别执行
- 审查模式：审查员工检查执行员工的结果
- 流水线模式：规划员工、执行员工、验证员工形成流水线
- 并行模式：多个专业员工从不同角度给出结论

**功能范围：**

- 员工团队配置：配置参与协作的数字员工团队
- 协作流程编排：定义协作流程和任务分派规则
- 子任务分派：将子任务分派给不同的数字员工
- 结果聚合：将多个员工的执行结果聚合
- 冲突处理：处理多个员工结果之间的冲突
- 人工仲裁：当自动冲突处理失败时，由人工仲裁


## 6. 能力管理

### 6.1 AI 模型管理

模型管理用于统一管理企业可用的 AI 模型供应商、模型配置和调用策略。

**功能范围：**

- 模型供应商管理：支持多家模型供应商和本地部署模型
- 模型目录：维护模型名称、能力标签、上下文长度、成本、延迟、适用场景等信息
- 凭证管理：管理 API 密钥、组织 ID、项目 ID、代理地址等凭证信息，支持密钥轮换
- 路由策略：按数字员工、任务类型、成本预算、可用性选择合适的模型
- 降级策略：主模型失败后切换备用模型
- 配额控制：按租户、部门、数字员工、任务限制 token 用量和费用
- 调用审计：记录模型、输入摘要、输出摘要、token 用量、耗时、错误和成本

**第一阶段范围：** 支持基础模型配置、调用和审计。复杂模型路由和成本优化可以后置。

### 6.2 工作节点管理

工作节点是数字员工执行真实任务的基础设施。

**功能范围：**

- 镜像管理：管理容器镜像
- 节点注册：注册新的工作节点
- 节点健康检查：定期检查节点是否可用
- 节点资源容量统计：统计节点的资源使用情况
- 节点标签：为节点打标签用于调度
- 可调度状态：设置节点是否可接受新任务
- 沙箱生命周期管理：管理沙箱的创建、启动、停止、销毁
- 工作目录管理：管理任务执行的工作目录
- 网络策略：配置节点的网络访问策略
- 凭证注入：安全地注入执行所需的凭证
- 命令执行审计：记录所有命令执行用于审计

**关键要求：**

- 数字员工不能直接在宿主机执行任务
- 所有代码执行、文件操作、命令运行都应发生在受控沙箱中
- 沙箱需要记录执行轨迹，便于审计、回放和问题定位

### 6.3 技能管理

平台支持三种类型的能力：工具（Tool）、技能（Skill）和外部服务（MCP）。

**工具（Tool）：** 提供基础的外部系统交互能力，例如代码查询、文件操作、网络请求等。工具通常粒度较小，是技能的组成部分。

**技能（Skill）：** 预定义的复杂能力组合，例如代码审查、文本摘要、分类判断等。技能可以直接被数字员工调用，通常由多个工具和一个执行流程组成。

**外部服务（MCP）：** 外部系统提供的服务能力，通过标准协议接入平台。

**管理功能：**

- 能力注册：注册新的工具、技能或外部服务
- 能力分类：按功能领域对能力进行分类
- 能力模板：提供能力开发的模板和示例
- 版本管理：管理能力的版本和变更
- 依赖声明：声明能力的依赖关系
- 权限声明：声明能力需要的权限
- 安全审核：对能力进行安全审核
- 启用和停用：控制能力的可用性


## 7. 知识库

知识库为数字员工提供企业内部上下文信息。

**功能范围：**

- 知识导入：从文档、代码仓库、数据库等来源导入知识
- 知识分类：按领域、项目、部门等维度分类知识
- 知识检索：支持数字员工查询相关知识
- 权限控制：控制不同数字员工和用户可访问的知识范围
- 知识更新：支持知识的增量更新
- 使用统计：统计知识的使用情况

**典型应用场景：**

- 代码仓库知识：帮助数字员工理解代码结构和规范
- 业务文档知识：帮助数字员工理解业务流程和规则
- 技术规范知识：帮助数字员工遵循企业的技术规范
- 历史案例知识：帮助数字员工参考历史解决方案


## 8. 企业级治理

企业级平台必须从一开始考虑治理能力。

### 8.1 多租户隔离

- 租户管理：支持多个企业租户
- 数据隔离：不同租户的数据完全隔离
- 资源隔离：不同租户的资源独立分配

### 8.2 用户和权限管理

- 用户管理：管理平台用户
- 组织管理：管理企业的组织和部门结构
- 部门管理：管理部门信息和层级关系
- RBAC 权限：基于角色的权限控制
- 能力权限：控制谁能使用哪些能力

### 8.3 审计能力

- 配置变更审计：记录数字员工配置的变更历史
- 任务执行审计：记录所有任务的执行过程和结果
- 工具调用审计：记录所有工具的调用情况
- 模型调用审计：记录所有模型的调用情况
- 执行回放：支持回放任务的执行过程

### 8.4 成本管理

- 成本统计：按租户、部门、数字员工、任务统计成本
- 预算限制：设置成本预算告警
- 异常告警：成本异常时告警
- 成本优化：提供成本优化建议

### 8.5 数据安全

- 敏感数据脱敏：自动脱敏敏感数据
- 密钥托管：安全地托管 API 密钥等敏感信息
- 数据加密：对敏感数据进行加密存储
- 访问控制：严格控制对敏感数据的访问


## 9. MVP 阶段规划

### 9.1 MVP 目标

第一版聚焦"代码助手数字员工"，实现 PR 自动审查的端到端流程。

**目标场景：**

1. 开发者在代码平台创建 PR
2. 数字员工被自动触发
3. 数字员工拉取代码变更
4. 数字员工在工作环境中分析代码
5. 数字员工生成审查结果
6. 数字员工回写审查评论到 PR
7. 平台保存任务记录和执行轨迹

### 9.2 MVP 功能清单

**数字员工基础：**

- 创建和配置数字员工
- 基础信息（名称、头像、描述、部门、岗位）
- 角色配置（职责范围、工作边界）
- 能力配置（默认模型、可用技能、工作节点）

**绑定关系：**

- 外部连接：支持代码平台（GitHub/GitLab）的通知接入
- 工作节点：支持容器沙箱执行环境

**执行能力：**

- AI 模型调用
- PR 审查技能
- 代码平台评论回写

**任务管理：**

- 任务创建和执行
- 任务状态跟踪
- 任务执行记录

**审计记录：**

- 任务执行记录
- 模型调用记录
- 工作节点命令记录

### 9.3 MVP 验收标准

- 可以创建一个代码助手数字员工
- 可以绑定模型、技能和容器工作节点
- 可以通过代码平台 PR 事件触发任务
- 可以在工作节点中执行代码分析
- 可以回写 PR 审查评论
- 可以查看任务执行记录
- 审查质量达到人工审查的 60% 以上
- 端到端延迟小于 30 秒
- 系统稳定性大于 99%
